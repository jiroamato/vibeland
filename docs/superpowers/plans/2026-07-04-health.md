# Health Implementation Plan (Survival v1, slice ④)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 10-heart health bar, vanilla fall damage, damage flash, death screen with inventory scatter, respawn at world spawn. Survival-only.

**Architecture:** Pure `health.ts` (Health class + `fallDamage`), fall-distance tracking inside `Player.update` exposed as a per-frame `landedFall` field, hearts + flash + death overlay in `ui.ts`/`index.html`, and `main.ts` glue (damage application, death scatter, respawn). Creative never constructs a Health.

**Tech Stack:** TypeScript, Vite, vitest, DOM.

## Global Constraints

- Conventional Commits; creative byte-for-byte unchanged; pure modules run in plain node; suite + `tsc --noEmit` + build stay green.

---

### Task 1: `health.ts` — Health class + fallDamage (TDD)

**Files:** Create `src/health.ts`; Test `test/health.test.ts`.

**Produces:** `MAX_HP = 20`; `class Health { hp; dead; damage(n); heal(n); tick(dt): boolean; reset() }`; `fallDamage(dist): number`.

Tests first: damage clamps at 0 and sets `dead`; heal clamps at MAX; `tick` heals 1 HP per 4 s only below max (returns true exactly on the healing tick); damage resets the regen interval; `reset()` restores 20; fallDamage table `[0→0, 3→0, 3.5→1, 4→1, 10→7]`. Implementation per the spec (`ceil(dist - 3 - 1e-6)` clamped ≥ 0; regen timer accumulates in `tick`, zeroed by `damage`).

Commit: `feat(health): Health model + vanilla fallDamage`

### Task 2: fall tracking in `Player`

**Files:** Modify `src/player.ts`.

- Fields: `landedFall = 0` (public, valid for one frame), `private fallDistance = 0`.
- In `update`, record `startY = this.pos.y` before integration and `wasAirborne = !this.onGround && !this.flying`. After integration: `this.landedFall = 0`; if flying → `fallDistance = 0`; else if the feet block is liquid (`blockDef(world.getBlock(...pos)).liquid`) → `fallDistance = 0`; else if `pos.y < startY && !this.onGround` → `fallDistance += startY - pos.y`; else if landing this frame (`this.onGround && fallDistance > 0`) → also add this frame's drop (`startY - pos.y` when positive), then `landedFall = fallDistance; fallDistance = 0`.
- No unit test (Player needs a full World); covered by the e2e fall in Task 4. Suite must stay green.

Commit: `feat(player): track fall distance, expose landedFall on touchdown`

### Task 3: hearts row, damage flash, death overlay (UI)

**Files:** Modify `src/ui.ts`, `index.html`.

- `#hearts` above `#hotbar` (same centring), 10 `.heart` spans, each a dark ♥ with a red ♥ overlay clipped to 100%/50%/0% width; `UI.setHealth(hp)` repaints; `UI.showHearts` flag (survival only).
- `#damageFlash`: full-screen `pointer-events:none` red radial vignette, `opacity 0`, `.hit` class + transition; `UI.damageFlash()` retriggers it.
- `#deathScreen` (z-index above pause overlay): dark red backdrop, "You died", `#respawnBtn` styled like the play buttons; hidden by default.

Commit: `feat(ui): hearts bar, damage flash, death screen`

### Task 4: wiring in `main.ts` + e2e

**Files:** Modify `src/main.ts`; README controls/architecture rows.

- `choose('survival')`: `health = new Health(); ui.showHearts = true; ui.setHealth(health.hp)`.
- Frame loop (inside the survival branch): apply `fallDamage(player.landedFall)` → `health.damage`, `ui.damageFlash()`, `ui.setHealth`, `die()` at 0; `health.tick(dt)` repaints hearts when it heals.
- `die()`: flush the open inventory screen (overflow → drops), scatter every inventory slot via `drops.spawn` with ±0.5 jitter, clear slots, `syncHotbar()`, show `#deathScreen`, `exitPointerLock`. `onLockChange` keeps the pause overlay hidden while the death screen is up.
- `#respawnBtn` click: `player.spawn(0,0)`, `health.reset()`, `ui.setHealth`, hide death screen, `input.requestLock()`.
- E2E: scripted fall (teleport +12 blocks, watch hearts drop by `fallDamage(12−ε)`), water landing safe, fall-to-death → scatter + death screen → respawn at spawn with 20 HP, creative regression (no hearts, flying cliff dives harmless).

Commit: `feat(health): fall damage, death scatter and respawn wiring` + `docs: health controls/architecture`
